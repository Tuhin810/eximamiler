# DNS records for the Exim sending node

For good deliverability every sending domain needs SPF, DKIM, DMARC, and
the VPS needs correct forward/reverse DNS. Replace `yourdomain.com`,
`mail.yourdomain.com`, and `203.0.113.10` with your real values.

## 1. A / PTR (reverse DNS) — required for direct-mx
| Type | Host | Value |
|------|------|-------|
| A    | `mail.yourdomain.com` | `203.0.113.10` |
| PTR  | `10.113.0.203.in-addr.arpa` | `mail.yourdomain.com` |

> PTR is set at your VPS provider, not your DNS host. Many MX servers
> reject mail whose PTR doesn't resolve back to the HELO name.

## 2. SPF
| Type | Host | Value |
|------|------|-------|
| TXT  | `yourdomain.com` | `v=spf1 a mx ip4:203.0.113.10 ~all` |

## 3. DKIM
Generated by `scripts/gen-dkim.sh` (run automatically on first boot).
Grab the printed record, e.g.:

| Type | Host | Value |
|------|------|-------|
| TXT  | `proxymail._domainkey.yourdomain.com` | `v=DKIM1; k=rsa; p=MIIBIjANBg...` |

Selector = `DKIM_SELECTOR` from `.env`.

## 4. DMARC
| Type | Host | Value |
|------|------|-------|
| TXT  | `_dmarc.yourdomain.com` | `v=DMARC1; p=none; rua=mailto:dmarc@yourdomain.com; fo=1` |

Start with `p=none` while warming up, then move to `quarantine` / `reject`.

## Verify
```sh
dig +short TXT proxymail._domainkey.yourdomain.com
dig +short TXT yourdomain.com
dig -x 203.0.113.10 +short          # PTR
```
Send a test to `check-auth@verifier.port25.com` and read the reply — it
scores SPF/DKIM/DMARC end-to-end.
